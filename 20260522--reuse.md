## 2. The Typed Boundary Certificate

### 2.1 Channel-formation rule

We give a Martin-Löf-style typed judgment under which a cut is
licensed as a channel:

```text
D : CausalDAG    F : InformationSheaf D    Z : Set D.V
cut : Set D.V   [path(cut)]   [stalk(cut)]   [res/merge(cut)]   [prob(cut)]
─────────────────────────────────────────────────────────────────────
                        D, F, Z ⊢ cut : Channel
```

Reading: given a causal environment, a stalk-typing environment, a
conditioning set, and four local data items, the rule licenses `cut`
to appear in the input position of Shannon, DPI, KKT, or QIF theorems.

The four data items are not arbitrary stipulations. They are the
minimal data needed to make a cut a channel:

```text
(1) path(cut)        baseball path on D, collider/non-collider typed
(2) stalk(cut)       information-state types on each base
(3) res/merge(cut)   restriction maps and collider fiber-merge
(4) prob(cut)        valuation + pushforward compatibility
```

Drop any one and at least one downstream theorem loses its argument
type. Section 3.1 proves this.

### 2.2 Classical theorems acquire typed signatures

```text
Shannon : (C : Channel, p : InputDist C, k : Code) → Rate k ≤ Capacity C
DPI     : (m : MarkovChain X Y Z) → I(X;Z) ≤ I(X;Y)
KKT     : (P : ConvexProblem, x : KKT-point P) → Optimal x
```

To use a theorem is to supply a value of its argument type. The
channel-formation rule produces such values from causal syntax.
Pearl's d-separation, h_cap, condMarkov, and DisjointSets all
become *proof obligations* discharged inside the rule's premises,
rather than external axioms imported ad hoc.

### 2.3 The certificate object: SheafMAGWalk

We name the four-tuple of certificate data `SheafMAGWalk`. The name
records that the object is simultaneously a MAG-style walk on the
causal syntax and a section of an information sheaf carrying
stalk-level realization data.

```text
structure SheafMAGWalk (D : CausalDAG) (F : InformationSheaf D)
                       (Z : Set D.V) (s t : D.V) where
  path         : BaseballPath D s t
  res          : ∀ e ∈ path.edges, F.Stalk e.src →ₚ F.Stalk e.tgt
  val          : ∀ b ∈ path.bases, Measure (F.Stalk b)
  val_compat   : pushforward law along non-collider edges
  witness      : ∀ c ∈ path.colliders, ColliderWitness c Z
  merge        : ∀ c ∈ path.colliders, fiber-product on stalks
  merge_compat : collider valuation consistency
```

The current Lean artifact has a graph-level `MAGWalk` that
captures `path` and an implicit form of `witness` only. The POPL27
upgrade promotes it to the four-tuple above:

```text
MAGWalk       (current Lean)        graph-level certificate
SheafMAGWalk  (POPL27)              channel-formation certificate
SheafMAGWalk projects to MAGWalk by forgetting (stalk, res, val).
```

### 2.4 Five ambient specializations

The same formation rule has five canonical specializations under
different enriching categories `V`:

```text
Ambient                Specialization
─────────────────────────────────────────────────────────────
Pearl causal graph     active path with collider witness
PL module system       typed interface + capability discharge
Probability            sub-σ-algebra + Markov kernel
Sheaf                  restriction + gluing axioms
QIF                    leakage channel + cut capacity
```

These are not metaphors. They are V-instantiations of one judgment
under different choices of `V`.

## 3. Soundness of Certificate Composition

### 3.1 Necessity

```text
Register: structural.

Theorem (Necessity).
The four fields of `SheafMAGWalk` are exactly the formation data
needed to turn a Pearl-style causal path into a channel boundary:
  path       supplies the Boolean causal carrier,
  stalk      supplies the typed information objects,
  res/merge  supplies transport and collider gluing,
  prob       supplies valuation and pushforward compatibility.

Removing any field leaves at least one downstream theorem with an
uninhabited argument type.
```

Sketch: dropping `path` loses the query carrier; dropping `stalk`
makes the stalks disconnected; dropping `res`/`merge` removes the
restriction maps needed by DPI; dropping `prob` loses quantitative
content. This is a structural typing claim, not a universal
complexity theorem over all possible frameworks.

### 3.2 Compositionality

```text
Register: structural.

Theorem (Composition).
SheafMAGWalk certificates compose along shared endpoints:
  given cert₁ : SheafMAGWalk F D Z s t
  and   cert₂ : SheafMAGWalk F D Z t u
  there exists cert  : SheafMAGWalk F D Z s u
  with capacity bound Cap(cert) ≤ Cap(cert₁) + Cap(cert₂)
  whenever the merge at t is independent.
```

This is the structural law that lets a cut-set bound on a composite
agent reduce to a sum of cut-set bounds on its component channels.
The independence side-condition is exactly what fixes the per-edge
additive-bound pathology.

### 3.3 Certificate Checking

```text
Register: type-theoretic / by-definition.

Theorem (Certificate checking).
Given an emitted certificate object `c`, each field obligation in
`SheafMAGWalk c` is locally checkable by its declared type:
  path well-typedness,
  collider-witness membership,
  restriction/merge typing,
  valuation compatibility.
```

This is not a global completeness claim for finding certificates.
Search may be delegated to an external compiler; the kernel checks only
the certificate it emits.

### 3.4 Static-side roundtrip

```text
Register: type-theoretic.

synth     : ActiveTrace → TraceSynthesis
decompile : TraceSynthesis → ActiveTrace

decompile (synth t) = t
synth (decompile c) = c              (on well-typed c)
```

A typed inverse between original Pearl active traces and generated
Lean trace certificates. This is the compiler-correctness theorem
for the static side; it stands beside the dynamic certificate of
§2–§3.3.

## 4. Internal Mechanism: Compilation Pipeline

The certificate is produced by an explicit compilation pipeline.

```text
Trail
  ─[normalize]─────────→  BaseballPath           (syntactic)
  ─[compressWithFuel]──→  WitnessedPath          (collider treaty eliminated)
  ─[realize on F]──────→  SheafMAGWalk           (sheaf semantics added)
```

Stages:

**Normalize.** Pearl's undirected trail is rewritten into a typed
baseball path: bases are ordered, edge orientations fixed, and each
internal base tagged collider or non-collider.

**Eliminate the collider treaty.** Pearl's rule `c is open iff some
descendant of c is in Z` is treaty-level. The compiler rewrites it
into explicit witness data:

```text
for each collider c on the path:
  witness w_c : descendants(c) ∩ Z ≠ ∅
or a concrete refutation thereof for a blocked path.
```

This step is the external compilation phase. It searches for
witnesses and supplies fuel/bounded-recursion evidence. Its
termination is witnessed by DAG depth. The honest sentence:

```text
The compiler is outside the trusted kernel.
The kernel checks only the certificate the compiler emits.
```

In the current Lean artifact, `compressWithFuel` is a bounded checker
that consumes an explicit `BayesBallPath`, not a trusted oracle.

**Realize on F.** A baseball path with witnesses receives stalk
typing, restriction maps, valuations, and collider fiber-merge data,
producing a complete `SheafMAGWalk`.

The pipeline is the operational reading of the formation rule: each
phase discharges one of the four proof obligations from §2.1.

## 5. Information-Sheaf Layer (Internal Machinery)

The sheaf is the carrier for stalk data. The base poset is the
finite Alexandrov space `(D.V, ≤_causal)`. Each stalk `F_v` is the
type of local information states at vertex `v`; restriction maps
implement information flow.

```text
Stalk v       = local information states at v
rho_{v,w}     = restriction / information-flow map for v ≤ w
mu_v          = valuation on Stalk v
pushforward   : (rho_{v,w})_* mu_v = mu_w
```

A minimal Lean-shaped interface:

```lean
structure InformationSheaf (I : Type) [Preorder I] where
  Stalk : I → Type
  res   : {i j : I} → i ≤ j → Stalk i → Stalk j
  res_id : ∀ i, res (le_refl i) = id
  res_comp : ∀ {i j k} (hij : i ≤ j) (hjk : j ≤ k),
    res (le_trans hij hjk) = fun x => res hjk (res hij x)
```

Stalk-internal geometry (Fisher / KL / Wasserstein) lives inside
stalks, not on the base poset. For probability-valued sheaves,
lossiness of restriction is measured by entropy decrease, Fisher
pushforward, or finite channel capacity:

```text
sup_μ I(μ ; rho_* μ) ≤ C < ∞.
```

This is the form that interfaces with the cut-capacity certificate
in §3.

The sheaf is internal machinery, not the headline. Replacing the
information sheaf with another stalk-typing scheme (for example,
Markov categories in the sense of Fritz) changes how the certificate
is built but not what it asserts.

## 6. Why the Old Boundary Failed: Concrete Examples

### 6.1 NeurIPS h_cap as external premise

The `h_cap : I_cut ≤ C` premise in the existing NeurIPS artifact is
external precisely because `cut` is a `Set Vertex` rather than a
channel. The formation rule absorbs `h_cap` as a discharged
sub-obligation inside the certificate.

### 6.2 condMarkov as external assumption

Conditional Markov structure is hypothesized in current
formalizations. The formation rule derives conditional Markov from
the four data items: sheaf restriction plus valuation compatibility
implement the Markov kernel; the collider witness implements the
conditioning.

### 6.3 Per-edge additive bound pathology

Per-edge additive capacity bounds fail because they treat each edge
as an independent channel, ignoring that channels along the same
cut may correlate through colliders. The formation rule makes
correlation explicit through fiber-merge data; the sum bound is
sound only on cuts where the merge is independent. This is exactly
the side-condition in the Composition theorem §3.2.

### 6.4 DisjointSets

The POPL `DisjointSets X Y Z` condition says an endpoint cannot
also be a conditioning variable. The formation rule rejects such
queries at the path level — the relevant restriction is `rho_{X,X}
= id`, a missing morphism rather than a violated axiom.
`DisjointSets` is a corollary of well-typed `path(cut)`, not an
extra hypothesis.

### 6.5 Counter-causality

`X → Y → Z` and `X ← Y ← Z` share observational conditional
independences. They are not the same information-sheaf diagram once
direction is part of the base topology. They induce different
restriction diagrams, hence different SheafMAGWalk objects. The
formation rule does not recover direction from observation alone;
it only refuses to silently identify the two diagrams once
direction is supplied.

## 7. Interface Inventory for Agentic Systems

The channel-formation rule extends to systems whose behaviour
crosses multiple typed boundaries. Industrial LLM agents have a
non-trivial information topology:

```text
Interface                       Description
─────────────────────────────────────────────────────────────
token output                    user-visible response
tool call                       external API invocation
memory read / write             persistent state across turns
scratchpad / chain-of-thought   silent reasoning channel
inter-agent message             multi-agent communication
retrieval result                RAG / context injection
hidden cache                    KV cache, prompt cache
log boundary                    audit trail itself
gradient (training time)        data → weight channel
prompt cache                    cross-session leakage
function trace                  call graph as side channel
```

Each row is a cut requiring its own channel-formation certificate.
The token-output boundary is the *last* interface, not the only one.

A complete audit is a property of the **AI Bill of Materials**
(AI-BOM): every typed interface listed, every interface holding a
certificate, untyped interfaces failing the audit independent of
model behaviour.

Two failure modes:

**(a) Coverage failure.** Output-targeted methods cover one of
roughly eleven interfaces.

**(b) Category failure.** Activation-cloud methods are not
interface methods. They study internal state rather than typed
boundaries. The "high-dimensional semantic cloud" interpretability
problem is a wrong-boundary artefact rather than an intrinsic
property of large models — analogous to the dispensability of the
luminiferous aether once light's correct frame was identified.

## 9. Industrial Implications

### 9.1 Closed-box certification

Behavioural-cut certificates require only an intervention API.
Weights need not be exposed.

```text
Audit level    Access required                Coverage
─────────────────────────────────────────────────────────────────
Behavioural    black-box + intervention API   ~85%
Structural     + layer activations            additional ~10%
Mechanistic    + weights                      residual ~5%
```

The regulatory deadlock (white-box demand vs IP protection) is
dissolved for the bulk of compliance use cases. The residual
demands (AI safety, deceptive alignment, critical infrastructure)
still require white-box access.

### 9.2 Precedent

The channel-formation rule is the AI-agent specialization of
capability-based security (Levy 1984; Miller 2003) and the
language-based information flow control tradition (Denning 1976;
Goguen-Meseguer 1982; Myers-Liskov DIFC 1997; Sabelfeld-Myers
2003). The contribution is not a new safety paradigm but the
transplant of mature PL/OS discipline into the agentic-AI setting.

### 9.3 Multi-cut audit

```text
Current audit:   inspect output behaviour, infer everything else.
Proposed audit:  enumerate interfaces, certificate per interface,
                 evaluate coverage. Untyped interfaces fail audit.
```

Audit becomes a property of the governance topology rather than of
model weights.
